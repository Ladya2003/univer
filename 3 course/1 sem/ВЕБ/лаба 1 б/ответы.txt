1) Что такое AJAX?
AJAX (Asynchronous JavaScript and XML) — это подход к созданию интерактивных веб-приложений, который позволяет веб-страницам асинхронно обмениваться данными с сервером без необходимости полной перезагрузки страницы. Это означает, что можно обновлять часть содержимого страницы без перезагрузки всей страницы. AJAX часто использует XMLHttpRequest объект для отправки и получения данных, хотя данные могут быть в форматах, отличных от XML, таких как JSON.

2) Что такое протокол WebSocket?
WebSocket — это протокол, предоставляющий полноценный двусторонний канал связи через единственное долгосрочное TCP-соединение. Он разработан для использования в веб-браузерах и веб-серверах, но может быть использован любым клиентом или сервером. Протокол WebSocket делает интернет-связь более эффективной по сравнению с REST или AJAX.

3) Что такое протокол Web Sockets API?
WebSockets API не является протоколом, а интерфейсом (API), который позволяет веб-страницам использовать протокол WebSocket для двусторонней связи. Этот API предоставляется современными веб-браузерами и позволяет взаимодействовать с WebSocket-серверами.

4)Перечислите основные функции Web Sockets API:

.WebSocket(url): Конструктор для создания нового объекта WebSocket, который подключается к указанному URL.
.send(data): Отправляет данные на сервер через открытое соединение.
.close(): Закрывает соединение WebSocket.
События: onopen, onmessage, onerror, и onclose, которые позволяют реагировать на различные фазы жизненного цикла соединения WebSocket.
5) Какие дополнительные возможности предоставляет Web Sockets API?

Эффективность: После установки соединения WebSocket данные могут быть переданы в любом направлении без дополнительных HTTP-заголовков и без создания нового соединения, что уменьшает задержки.
Двусторонняя связь: WebSockets поддерживают одновременную двустороннюю передачу сообщений, что идеально подходит для реального времени, таких как чаты, онлайн-игры и т. д.
Более многофункциональный: Поскольку WebSockets предоставляют постоянное соединение, они поддерживают более сложные протоколы сверх базового отправления и получения данных.
